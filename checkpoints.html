Checkpoint Strategy

Structure up to three monetized checkpoints

Mix providers, shortlinks, and captchas while keeping proof tokens secure.

Recommended mix

Shortlink → Ad view → Captcha (fallback ready)

Checkpoint plan

Each checkpoint issues a one-time proof token with a short expiration.

Checkpoint 1

Shortlink
  • Primary: shortlink or lightweight ad
  • Proof token TTL: 2 minutes
  • Fallback: captcha

Checkpoint 2

Ad view
  • Primary: ad view or offerwall
  • Detect adblock and swap providers
  • Track completion rate per provider

Checkpoint 3

Captcha
  • Primary: captcha challenge
  • Cooldown enforced server-side
  • Token bound to session fingerprint

Provider operations

Stay agile by rotating providers and observing completion rates.

Rotation rules

Swap providers if fill rate < 85% or drop-off spikes.

Fallback ladders

Define a secondary and tertiary provider per checkpoint.

Quality checks

Validate completion tokens against provider callbacks.
