Endpoint Spec

Minimum API surface

Keep the endpoint set small, secure, and easy to monitor.

Base URL

https://api.yourdomain.com

Endpoint groups

Group endpoints by session, checkpoint, and key lifecycle.

Session

  • POST /api/session/start
  • GET /api/session/status?sessionId=
  • POST /api/session/refresh
  • GET /api/config

Checkpoint

  • GET /api/checkpoint/next
  • POST /api/checkpoint/complete
  • POST /api/checkpoint/verify

Key

  • GET /api/key
  • POST /api/key/validate
  • POST /api/key/revoke
  • GET /api/health

Example payload

Use consistent response shapes for easier client handling.

POST /api/checkpoint/complete

{
  "sessionId": "sess_6d9c",
  "checkpoint": 2,
  "provider": "shortlink",
  "proofToken": "proof_abc123"
}

Response

{
  "ok": true,
  "nextCheckpoint": 3,
  "nextUrl": "/checkpoint/3",
  "expiresIn": 120
}

POST /api/key/validate

{
  "key": "KEY-ABC123"
}

Response

{
  "ok": true,
  "valid": true,
  "key": "KEY-ABC123"
}
