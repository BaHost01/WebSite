Anti-bypass

Protect sessions, checkpoints, and keys

Layer defenses to prevent replay attacks, automation, and token tampering.

Core principle

Trust server-side proofs, never client-side progress.

Defense layers

Stack multiple controls so bypass attempts fail at each step.

Session binding

Bind sessions to fingerprint + IP hash. Rotate tokens every 10 minutes.

Signed proofs

Use HMAC/JWT tokens with nonce + short TTL, redeem once.

Rate limits

Throttle by IP, fingerprint, and device to stop rapid retries.

Captcha fallback

Trigger captcha on adblock detection or suspicious behavior.

Key expiration

Keys expire quickly and require validation on executor startup.

Audit logs

Track failed proof validations and session anomalies.

Alerting checklist

Get notified when traffic or failures spike.

Failed proofs

Alert if proof validation errors exceed baseline.

Session spikes

Alert on sudden session creation surges.

Key retries

Flag excessive key requests per fingerprint.
